# 项目文档风格指南

本文档定义了 zyDocs 技术博客的写作风格和规范。所有新文档应遵循此指南以保持风格一致性。

## 1. 文档结构

### 标准结构模板

```markdown
# 标题

## 什么是 xxx？
简要概念介绍，用通俗易懂的语言解释。

::: tip
一句话总结核心要点。
:::

## 核心概念/特性
- **要点1**：解释说明
- **要点2**：解释说明

## 实操/代码示例
具体实现步骤和代码。

## 最佳实践/注意事项
::: tip
重要提示或建议。
:::
```

### 章节层级
- `#` - 文档主标题（只能有一个）
- `##` - 主要章节
- `###` - 子章节
- 避免使用四级及以下标题

## 2. 语言风格

### 语调
- **简洁直接**：避免冗长描述，直击要点
- **口语化**：像与人对话一样自然，可适当使用"简单来说"、"让我们"等表达
- **实用导向**：重在"怎么做"，而非纯理论
- **中文为主**：面向中文读者，技术术语可保留英文

### 示例
```markdown
✅ 简单来说，MCP 让大模型能够安全、可控地调用外部工具。
❌ MCP（Model Context Protocol）是一种开放协议，它通过标准化模型与外部资源的交互方式...

✅ 下载 vscode 插件 Cline
❌ 请前往 Visual Studio Code 应用商店下载并安装名为 Cline 的扩展程序
```

## 3. 格式规范

### 技术术语
- 使用**反引号**包裹：`MCP`、`LLM`、`API`、`function`
- 首次出现可写全称，后续用缩写：`Model Context Protocol（MCP）`
- 文件路径、变量名、函数名都用反引号：`config.mts`、`baseURL`

### 强调方式
- **加粗**：用于重要概念、关键词
- `代码格式`：用于技术术语、变量、路径
- > 引用：用于补充说明、示例场景

### 列表使用
- 无序列表（`-`）：用于列举同等重要的项
- 有序列表（`1.`）：用于步骤说明
- 列表项内可用 **加粗** 突出关键词

```markdown
- **标准化集成**：统一接口连接各种服务
- **功能扩展**：赋予实时数据访问能力
```

## 4. 代码规范

### 代码块
- 必须标注语言：` ```js `、` ```bash `、` ```json `
- 代码前简要说明其功能
- 关键行可添加注释

```markdown
前端调用示例：
```js
const response = await fetch(endpoint, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ message: '你好' })
});
```
```

### 行内代码
- 短变量、属性名用行内代码：`stream`、`collapsed`
- 命令行指令用行内代码：`npm install`、`git push`

## 5. 提示框使用

VitePress 提示框类型及使用场景：

```markdown
::: tip
提示信息 - 补充说明、小技巧、优化建议
:::

::: warning
警告信息 - 注意事项、潜在问题
:::

::: danger
危险信息 - 严重警告、可能出错的操作
:::

::: note
备注信息 - 额外的背景知识
:::
```

### 使用原则
- `tip` 最常用，用于强调重点
- 提示框内容应简洁，一行或两行
- 不要在提示框内嵌套代码块

## 6. 图片规范

### 路径格式
- 本地图片：`../../../public/xxx.png`
- 外部图片：直接使用完整 URL

### 使用场景
- 架构图、流程图
- 操作截图（配合步骤说明）
- 示意图

### 示例
```markdown
<img src="../../../public/mcp.png">

或者

![描述文字](../../../public/xxx.png)
```

## 8. 完整示例文档

参考以下模板创建新文档：

```markdown
# 文档标题

## 什么是 xxx？

一句话定义，用 `技术术语` 包裹专业词汇。

::: tip
一句话总结核心价值。
:::

## 核心特性

- **特性1**：简要说明
- **特性2**：简要说明
- **特性3**：简要说明

## 安装/准备

```bash
npm install xxx
```

## 代码示例

```js
// 示例代码
const result = await api.call();
```

## 注意事项

::: tip
重要提示内容。
:::

## 相关资源

- [官方文档](https://example.com)
- [工具推荐](https://example.com/tool)
```

## 9. 目录组织

- 速记/随笔：直接放在 `blog/myself/xxx/` 下
- 系列教程：创建 `chapter-01/`、`chapter-02/` 目录
- 每个目录下必须有一个 `index.md` 作为入口

## 10. 侧边栏配置

新增文档后，需要在 `.vitepress/config.mts` 中添加侧边栏链接：

```typescript
{
  text: '前端角度的AI',
  collapsed: true,
  items: [
    { text: '显示文字', link: '/blog/myself/ai/文件名.md' },
    // ...
  ]
}
```
